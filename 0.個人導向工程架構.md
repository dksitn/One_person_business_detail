# 🧩 Modular AI System — 個人導向工程架構

> 一套為「個人開發者」設計的 AI 系統架構，  
> 結合工程邏輯與創作流程，讓你可以快速實驗、整理、上線。  
>  
> 🌱 適合：Colab + VSCode + Railway 一人開發者  
> ⚙️ 特點：工程化結構、自由模組化、單人閉環循環  

---

## 🧭 一、架構設計原則

| 原則 | 意義 | 實際操作 |
|------|------|----------|
| 🧠 **以自己為主要使用者** | 架構為效率設計，不為團隊分工 | 資料夾幫你記得放哪裡，不限制創作 |
| 🧩 **工程化但可自由變形** | 有標準層級，但不必全開 | 想快寫就快寫，穩定後再模組化 |
| 🔁 **單人閉環設計** | Colab → VSCode → Railway 自用循環 | 自己開發、自己部署、自己使用 |
| 🌱 **自然長成的模組** | 功能重複才抽出共用邏輯 | `core/` 與 `shared/` 會自然長大 |
| 🎯 **以專案為核心單位** | 每個 idea 一個資料夾 | 專案是系統的主要單位 |

## 🧩 三、層級說明（簡明版）

### 1️⃣ `projects/` — 專案層
> 放你所有具體創作與實驗主題。  
> 每個資料夾是一個完整 idea，可自由混程式、Prompt、資料。

📘 範例：
```

projects/chatbot/
├─ prompts/chat_logic.md
├─ scripts/reply_engine.py
└─ demo_app/app.py

```

---

### 2️⃣ `core/` — 共用邏輯層
> 把你常用的功能集中在這裡，變成共用的 AI 腦袋。  

📘 範例：
```

core/adapters/llm_adapter.py    ← 統一呼叫 AI 模型
core/utils/text_cleaner.py      ← 清理使用者輸入
core/bundles/report_builder/    ← AI 自動生成報告的主流程

```

---

### 3️⃣ `shared/` — 共用素材層
> 放 Prompt 模板、通用工具、設定檔。  
> 像 LEGO 零件櫃，一次做，永久重用。

📘 範例：
```

shared/base_prompts/system_policy.md
shared/utils/logger.py
shared/settings/env_template.json

```

---

### 4️⃣ `dev/` — 開發與實驗層
> 一切新想法與實驗都從這裡開始。  
> 可以亂，能追蹤，就好。

📘 範例：
```

dev/notebooks/test_prompt.ipynb
dev/scripts/test_parser.py
dev/sandbox/ai_poem_game/
dev/tools/build_prompt.py

```

---

### 5️⃣ `app/` — 運行層
> 放要部署到 Railway 的最終版本。  
> 由 `dev/tools/` 自動生成成品。

📘 範例：
```

app/api/main.py
app/code/pipeline.py
app/prompts/final_prompt.md
app/microapps/gem_tester/app.py

````

---

### 6️⃣ `registry/` — 登錄層
> 管理版本、模組依賴與更新紀錄。  
> 就像你的系統「履歷表」。

---

### 7️⃣ `docs/` — 文件層
> 放筆記、方法論與開發流程。  
> 幫未來的你記得「這套系統怎麼長出來的」。

---

## 🔁 四、開發與部署流程

```mermaid
graph TD
A[Colab / dev/] --> B[projects/ - 寫功能與 Prompt]
B --> C[core/ or shared/ - 抽出重複邏輯]
C --> D[tools/build_code.py - 組裝成 app/]
D --> E[Railway - 自動部署雲端版本]
E --> F[雲端網站運行 🚀]
````

## 🧱 五、這樣設計對個人開發的好處

| 問題           | 傳統做法         | 新架構解法                      |
| ------------ | ------------ | -------------------------- |
| 想法多、程式亂      | 全塞在同個資料夾     | 每個 idea 一個 `project/`      |
| 想重用功能        | 複製貼上、容易版本錯   | 放進 `core/` 共享使用            |
| Prompt 太多找不到 | 分散在筆記中       | 集中於 `shared/base_prompts/` |
| 上線麻煩         | 手動上傳 Railway | 用 `tools/` 自動組裝部署          |
| 想測新東西        | 只能開新 repo    | 在 `dev/sandbox/` 測試即可      |
